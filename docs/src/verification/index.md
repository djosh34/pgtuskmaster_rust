# Verification

This section records how we verify that documentation matches the repository source-of-truth.

It is intentionally explicit and skeptical:
- Every behavior claim is assumed wrong until proven.
- “Verified” claims must have concrete evidence anchors (code or tests).
- “Absence” claims (“never”, “does not”, “cannot”) must have a guard or a test, or they are rewritten as bounded guidance.
