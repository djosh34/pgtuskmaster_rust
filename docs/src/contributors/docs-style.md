# Docs Authoring

Contributor-facing documentation should be accurate, traceable, and easy to maintain as behavior evolves.

## Writing principles

- lead with reader intent before internals
- keep terminology consistent across sections
- explain behavior, then explain rationale and tradeoffs
- prefer short paragraphs with explicit transitions

## Content boundaries

- Operator sections avoid programming-language walkthroughs.
- Contributor sections may include focused code snippets where they clarify architecture.
- Diagrams should explain runtime behavior, not serve as decorative navigation.

## Consistency rules

- keep safety and trust terms aligned with lifecycle chapters
- keep API and config facts synchronized with canonical reference pages
- rewrite broad absolute claims unless guarded by code or tests
