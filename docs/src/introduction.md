# Introduction

This book documents **how to operate** and **how to reason about** the `pgtuskmaster` system:

- Start with [Start Here](./start-here/index.md) to understand the problem statement, the system’s approach, and how the book is organized.
- Use [Quick Start](./quick-start/index.md) to get a working single-node lab and learn how to interpret health signals.
- Use the [Operator Guide](./operator/index.md) and [System Lifecycle](./lifecycle/index.md) for day-2 operations and failure-mode reasoning.

If you are contributing code or tests, use [Contributors](./contributors/index.md) as the source of truth for implementation invariants and verification expectations.
